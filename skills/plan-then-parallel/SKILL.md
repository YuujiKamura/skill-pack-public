## 原則

1. **プランモードで計画を確定** - 曖昧なまま動かない
2. **イシューに分解** - 独立して実行可能な単位に
3. **並列発火** - Task toolで同時に投げる
4. **待たない** - 結果が返ったものから次へ

## 実行パターン

```
ユーザー要件
    ↓
EnterPlanMode（計画策定）
    ↓
ExitPlanMode（承認）
    ↓
TaskCreate × N（イシュー化）
    ↓
Task tool × N（並列実行、run_in_background: true）
    ↓
結果マージ
```

## 並列発火のコツ

- 単一メッセージ内で複数のTask toolを呼ぶ
- run_in_background: true で待機しない
- 依存関係があるものは直列、ないものは並列
- 失敗したタスクは捨てて先に進む

## アンチパターン

- 計画なしに並列発火 → 手戻り多発
- 全部直列 → 遅い
- 結果を全部待つ → ボトルネック
- 曖昧な指示で発火 → 成果物がバラバラ
