# PDF書類チェックスキル

PDFファイルの整合性をチェックし、ガイドラインを蓄積・活用する。

## ガイドライン

**グローバルガイドライン**（全プロジェクト共通）:
```
<CLAUDE_HOME>/skills/pdf-check/.guidelines.json
```

解析時は必ずこのガイドラインを読み込み、該当カテゴリの観点でチェックする。

## 実行手順

### 1. ガイドライン読み込み

```bash
cat <CLAUDE_HOME>/skills/pdf-check/.guidelines.json
```

### 2. 書類タイプ判定とガイドライン適用

| ファイル名キーワード | 書類タイプ | 適用ガイドライン |
|-------------------|-----------|-----------------|
| 契約 | 契約書 | categories.契約書 |
| 見積 | 見積書 | categories.見積書 |
| 温度, 温度管理 | 温度管理 | categories.温度管理 + categories.安全管理 |
| 交通誘導, 配置, 警備 | 交通誘導員 | categories.交通誘導員配置実績 |
| 写真, 写真帖 | 写真帖 | categories.写真帖 + categories.安全管理 |

### 3. 解析実行

**Claudeで直接解析**（推奨）:
- PDFをReadツールで読み込み
- ガイドラインの観点で視覚的内容を含めてチェック
- テキスト整合性 + 写真内の安全管理を確認

**ShoruiCheckerで解析**:
```bash
# ヘッドレスモード
<USER_HOME>/yuuji/ShoruiChecker/src-tauri/target/release/shoruichecker.exe --headless "$PDF_PATH"

# GUIモード
<USER_HOME>/yuuji/ShoruiChecker/src-tauri/target/release/shoruichecker.exe "$PDF_PATH"
```

### 4. チェック観点

**テキスト整合性**:
- 数値の一致（温度、金額、人数など）
- 日付・時系列の論理性
- 名称・場所の一貫性

**視覚的内容**（写真帖の場合）:
- 作業員の安全装備
- ヘルメットのあご紐（必須）
- 重機運転者の装備

### 5. 出力形式

```
## [書類タイプ]

### テキスト整合性
✓ 問題なし: [項目]
⚠ 問題あり: [項目] - [具体的な指摘]

### 安全管理（写真帖の場合）
✓ 問題なし: [項目]
⚠ 問題あり: [ページ、写真番号] - [具体的な指摘]
```

### 6. ガイドライン更新

新しい問題を発見したら、グローバルガイドラインに追記:

```bash
# 現在のガイドラインを確認
cat <CLAUDE_HOME>/skills/pdf-check/.guidelines.json

# 新しい観点を追加して更新
```

優先度の区分:
- **必須（指摘される）**: 必ず見られる項目
- **推奨（見逃されることもある）**: 時々スルーされる項目

## ガイドライン構造

```json
{
  "common": ["全書類共通の観点"],
  "categories": {
    "カテゴリ名": ["観点1", "観点2"],
    "安全管理": {
      "必須（指摘される）": ["あご紐確認"],
      "推奨（見逃されることもある）": ["安全チョッキ"]
    }
  }
}
```
