# 出来形正規化ルール

## 原則
- 出来形の測定値統一は `station` ではなく `photo-tagger` の `group` を唯一キーにする。
- `group` を再編成する補正（時刻再グループ・測点名再グループ）は行わない。
- 同名測点が存在しても、異なる `group` であれば別セットとして扱う。

## group=0 の扱い
- `group=0` の出来形写真は自動統一しない（誤統一を防ぐため）。
- 必要なら解析段階（photo-tagger）を再実行して `group` を付与してから正規化する。

## 表層の必須要件
- 表層出来形は `計画高` ベースで判定する。
- 出力は常に `V1-V5` と `幅員W1/W2` の両方を含める。
- 欠落がある場合はPDF化前に `result.json` を点検する。

## 混線防止チェック
- `No.1` と `取付道路 No.1` が同一値になっていないことを確認する。
- 測点ごとの確認は以下を最小セットにする。
  - 測点名
  - group番号
  - measurements 1行目
  - V1/V3/V5 と W1/W2 の主要値

## 検証手順（推奨）
1. `normalize` 実行後に `result.json` を測点別一覧化する。
2. 同名測点が複数ある場合、`group` 別に値が分離されていることを確認する。
3. PDF出力後、代表測点（No.1、取付道路 No.1）を目視確認する。
