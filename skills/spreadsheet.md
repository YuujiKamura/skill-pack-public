# スプレッドシート

会話数: 5 | パターン数: 5

## 1. win32comによる高精度なExcelテンプレート操作

書式やチャートを完全に保持したままシートをコピー・編集するために、標準ライブラリ（openpyxl等）ではなくWindowsのCOMインターフェースを直接操作するパターン。

### 手順

1. python -c "import win32com.client" で利用可能性を確認する
2. Templateシート（例：切削基準高V_横）のセル位置やレイアウトを特定する
3. win32comを使用してワークブックを開き、Templateを必要数コピーする
4. 各シートの名前を変更し、必要なデータをセルに書き込む
5. 変更を保存し、COMオブジェクトをクリーンアップする

出現頻度: 3回

## 2. 外部ソース（PDF/JSON）からの工事写真・試験データ統合

現場で撮影された写真（JSON化されたメタデータ）や試験結果PDFから、Excelの品質管理・出来形管理図表へデータを転記・集計するパターン。

### 手順

1. 工事写真まとめ（PDF）や材料確認（PDF）の中身を確認する
2. 温度管理や散布量などの実測データを抽出する
3. Excel内の実施記録や集計表と突き合わせ、数値を上書き・更新する
4. 写真と図表の整合性（温度の±判定など）をチェックする

出現頻度: 4回

## 3. 表示値ベースの整合性修正と数式デバッグ

Excel内の計算式（平均値、合計）のバグ修正や、表示されている値（丸め後）に基づいた判定基準への修正を行うパターン。

### 手順

1. 計算式のセル（平均値など）を確認し、バグを特定する
2. 「表示値ベース原則」（例：149.4℃は表示上149℃なので合格とする）に従い判定ロジックを修正する
3. 修正前後の整合性を検証するためのテンポラリファイル（temp_verification.txt等）を作成する

出現頻度: 2回

## 4. マルチフォーマット（XLS/XLSX）対応とライブラリ使い分け

古い形式（.xls）と新しい形式（.xlsx）が混在する環境で、適切なPythonライブラリを選択するパターン。

### 手順

1. ファイルの拡張子を確認する
2. .xlsxの場合は openpyxl を、.xlsの場合は xlrd を試行する
3. 読み取り専用か編集が必要か、書式維持が必要かに応じて win32com への切り替えを判断する

出現頻度: 3回

## 5. Pythonスクリプトによる一括自動処理

手動操作（MCPなど）の不安定さを避けるため、一連のExcel操作をPythonスクリプト（create_yoko_sheets.py等）として書き出し、一括実行するパターン。

### 手順

1. 操作内容を定義したPythonスクリプトをローカルに生成する
2. 絶対パス（r'...'）を使用してGoogle Drive上のファイルを指定する
3. run_shell_commandでスクリプトを実行し、結果をログで確認する

出現頻度: 3回
