# 目次スプレッドシート（インデックス）の設計思想

## 2つのスプレッドシートを混同するな（最重要）

SekouTaiseiMakerでは2つの異なるスプレッドシートを使用する。**絶対に混同するな。**

| スプレッドシート | 役割 | 含まれるシート | getActiveSpreadsheet() |
|-----------------|------|---------------|----------------------|
| **GASバインド先** | アプリのデータ保存用 | ProjectData, _Settings, _Templates | ✓ 返される |
| **目次スプレッドシート** | 工事情報の本体 | 工事内容, SS一覧, 施工体制 | ✗ 返されない |

### GASバインド先（アプリ保存用）

GASスクリプトがバインドされているスプレッドシート。`SpreadsheetApp.getActiveSpreadsheet()` で取得される。

- `ProjectData` シート: 業者一覧、書類ステータス等（アプリ内部データ）
- `_Settings` シート: APIキー、目次スプレッドシートID等
- `_Templates` シート: 書類テンプレート

### 目次スプレッドシート（工事情報本体）

工事ごとに作成される実際のデータ。**GASバインド先とは別のスプレッドシート。**

- `工事内容` シート: 工事基本情報（発注者、工期等）
- `SS一覧` シート: 関連スプレッドシートのリンク一覧
- `施工体制` シート: 業者ごとのスプレッドシートID

### 接続方法

GASバインド先の `_Settings` シートに目次スプレッドシートIDを保存し、必要に応じて `openById()` で開く。

```javascript
// 正しいアクセス方法
const settings = loadSettings();  // _Settingsから読み込み
const indexId = settings.indexSpreadsheetId;
const indexSS = SpreadsheetApp.openById(indexId);  // 目次を開く
const ssListSheet = indexSS.getSheetByName('SS一覧');
```

## 基本原則：目次スプレッドシートが全ての参照元

工事ごとに「目次スプレッドシート」（インデックス）を作成し、これを全ての情報の起点とする。

```
目次スプレッドシート（インデックス）
├── 工事内容シート ← 全ての工事基本情報を集約
│   ├── 契約日、契約金額
│   ├── 工事名称、工事内容、工事場所
│   ├── 工期（自・至）
│   ├── 発注者、担当課、TEL、地域情報
│   ├── 監督員名
│   └── 工事概要
│
└── 他の施工計画書・各種書類
    └── Linkシート → IMPORTRANGE で目次を参照
```

## 施工計画書のLinkシート構成

施工計画書などの各スプレッドシートには「Link」シートを設け、目次スプレッドシートからIMPORTRANGEでデータを取得する。

```python
# Linkシートの構成例
# A1: 目次スプレッドシートID
# A2以降: =IMPORTRANGE(A1, "工事内容!A1:C32")

# 設定例
updates = [
    {'range': 'Link!A1', 'values': [['目次スプレッドシートID']]},
    {'range': 'Link!A2', 'values': [[f'=IMPORTRANGE("{index_id}", "工事内容!A1:C32")']]}
]
```

---

## 他のシートからLinkシートを参照

各シート（表紙、１概要など）は、Linkシートの値をセル参照で使用する。

```python
# １概要シートの例
updates = [
    {'range': '１概要!C2', 'values': [['=Link!B4']]},   # 工事名称
    {'range': '１概要!C3', 'values': [['=Link!B6']]},   # 工事場所
    {'range': '１概要!C4', 'values': [['=Link!C7']]},   # 工期 自
    {'range': '１概要!C5', 'values': [['=Link!C8']]},   # 工期 至
    {'range': '１概要!C6', 'values': [['=Link!B10']]},  # 発注者
    {'range': '１概要!C8', 'values': [['=Link!B3']]},   # 契約金額
]
```

---

## メリット

1. **一元管理**: 目次を更新すれば全ての書類に反映される
2. **整合性**: 書類間のデータ不整合を防止
3. **効率化**: 各書類でベタ書きしない
4. **トレーサビリティ**: データの出所が明確

---

## 注意事項

- **目次スプレッドシートは直接編集しない**: 他の書類から参照されているため、構造を変えると参照が壊れる
- **IMPORTRANGEの初回アクセス許可**: 新しい書類から目次を参照する際は、手動でアクセス許可が必要
- **行番号の変動に注意**: 目次に行を挿入すると、参照先の行番号がずれる

## 参照乱れの修正方法

layoutシートなどでREGEXEXTRACTを使用している場合、参照先の行番号がずれることがある。

```python
# 修正例：layoutシートG24の参照修正
# 誤: =REGEXEXTRACT(Link!B13, "([^\s]+)$")  ← 地域情報を参照
# 正: =REGEXEXTRACT(Link!B14, "([^\s]+)$")  ← 監督員名を参照

sheets_service.spreadsheets().values().update(
    spreadsheetId=spreadsheet_id,
    range="'layout'!G24",
    valueInputOption='USER_ENTERED',
    body={'values': [['=REGEXEXTRACT(Link!B14, "([^\\s]+)$")']]}
).execute()
```

## IMPORTRANGEの自動更新

目次スプレッドシートを更新すれば、それを参照している他のスプレッドシート（Linkシート経由）は自動的に反映される。手動で各シートを修正する必要はない。

## データソースと表示先の区別（重要）

スプレッドシートには「データソース」と「表示先」の2種類がある。**混同するな。**

### データソース（マスターデータを保持）

| シート | 内容 |
|--------|------|
| 目次 → 工事内容シート | 工事基本情報（工事名、発注者、工期等） |
| 目次 → 施工体制シート | 業者ごとのスプレッドシートID一覧（B35, B37等） |
| 業者ごとの施工体制スプレッドシート → 施工体制シート | 業者情報（会社名、地域情報、TEL等） |

### 表示先（IMPORTRANGEで参照、印刷用書式）

| シート | 参照元 |
|--------|--------|
| 施工体制台帳 | 目次・業者の施工体制スプレッドシートをIMPORTRANGE |
| 下請契約書 | 目次・業者の施工体制スプレッドシートをIMPORTRANGE |
| 施工計画書 | 目次をLinkシート経由でIMPORTRANGE |

### GASでのデータ取得

- `getDisplayValues()` を使えばIMPORTRANGE展開後の値を取得可能
- 表示先からも値は取れるが、それは**参照結果**であり**ソース**ではない
- 照合の際は、ソースと表示先の両方を取得して比較することで整合性を確認できる

### 設計時の注意

1. **データソース一覧を作る際は、表示先を含めるな**
2. **データ修正は必ずソース側で行う**（表示先を直接編集してもIMPORTRANGEで上書きされる）
3. **ソースがどこか不明な場合、Linkシートを見ればIMPORTRANGE先が分かる**
