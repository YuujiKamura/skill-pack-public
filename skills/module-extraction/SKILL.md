# 巨大ファイルからのモジュール分離スキル

## 問題

5000行超のlib.rsなど巨大ファイルをサブエージェントに読ませると、コンテキストオーバーフローで処理が停止する。

## 対策ルール

### 1. Readツールのlimit制限

```
絶対厳守: limit=200 以下で読め
```

良い例:
```
Read lib.rs offset=2805 limit=200
Read lib.rs offset=3005 limit=200
```

悪い例:
```
Read lib.rs  # 全部読む - NG
Read lib.rs offset=0 limit=2000  # 多すぎ - NG
```

### 2. grepで構造把握（中身を読まない）

```bash
# 関数・構造体の一覧だけ取得
grep -n "^pub fn\|^fn\|^pub struct\|^struct\|^impl" lib.rs
```

### 3. 分割手順テンプレート

サブエージェントへの指示例:

```
**対象**: /path/to/lib.rs

**絶対厳守: Readツールは limit=200 以下で使え**

**手順**:
1. `Read lib.rs offset=XXX limit=200` で対象範囲を読む
2. 必要に応じて次の200行を読む
3. 新ファイル xxx.rs を作成（use文 + 構造体 + impl）
4. lib.rsから該当コードを削除
5. lib.rsに `mod xxx; pub use xxx::*;` 追加
6. `cargo check` でビルド確認
```

### 4. 分割優先順位

1. **依存なしモジュール優先** - data_model等、他に依存しない構造体
2. **ヘルパー関数** - add_line, add_text等の小さい関数群
3. **機能単位モジュール** - 図面生成、UI等
4. **アプリ本体** - 最後に残るメインロジック

### 5. ビルド確認は毎回

各モジュール分離後、必ず `cargo check` を実行。エラーがあれば即修正。
